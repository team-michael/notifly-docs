# 캠페인 대시보드

노티플라이의 캠페인 대시보드는 캠페인의 성과를 한 눈에 확인할 수 있는 화면입니다. 캠페인의 발송 현황, 전환 등을 확인할 수 있습니다.

## 1. 지표 설명 {#metrics}

### 1-1. 발송 성공 {#message-send-success}

노티플라이에서 유저의 메시지 주소 (전화번호, 푸시 토큰)로 발송을 성공적으로 한 경우입니다.

### 1-2. 발송 실패 {#message-send-failure}

노티플라이에서 유저의 메시지 주소로 발송을 실패한 경우입니다. (예: 잘못된 전화번호, 유효하지 않은 푸시 토큰 등)
친구톡의 경우 발송을 시도하였으나 발송 대상(고객) 중 현재 카카오 채널의 친구가 아닌 경우, 카카오톡을 사용하지 않는 경우 등에 대해서 발송 실패로 집계됩니다.

### 1-3. 수신 {#message-delivered}

유저의 기기로 발송이 도달한 경우입니다.

### 1-4. 오픈 {#message-open}

유저가 메시지를 열어본 경우입니다.

### 1-5. 클릭 {#message-click}

유저가 메시지 안의 링크 등을 클릭한 경우입니다.

### 1-6. 전환 {#message-conversion}

전환 이벤트를 설정하시면 사용자가 문자 혹은 푸시 등을 받은 이후 전환 기여 기간 내에 앱(혹은 웹)에 들어와 해당 이벤트를 발생 시키는 것을 집계합니다. 캠페인 별로 설정하실 수 있습니다. 설정 방법은 [노티플라이 캠페인 > 시작하기](/ko/user-guide/campaigns/basic#analytics)를 참고해주세요.

전환의 경우 클릭이 아닌, 메시지 발송 대상자들에 대한 전환을 집계합니다 ("간접 전환"). 예를 들어, 푸시를 오픈한 후 앱 종료 후 다시 직접 들어와 구매하는 경우에도 전환으로 집계합니다.
클릭을 하지 않아도 메시지를 받는 것 자체로 유저들이 영향을 받는다는 것을 고려한 방식입니다.

## 2. 채널별 지표 집계 {#channel-metrics}

- 앱 푸시, 웹 푸시: 발송 성공, 수신, 오픈, 클릭, 전환
- 앱 팝업, 웹 팝업: 노출, 클릭, 전환
- 이메일: 발송 성공, 수신, 오픈, 클릭, 전환
- 카카오 알림톡, 카카오 친구톡, 문자: 발송 성공, 전환
